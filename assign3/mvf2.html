Wish List - MVF 2 for OES website

Team member in charge of MVF - Marcus Lew



Introduction:

The idea of the wish list is to allow for users to pin their favourite items. This eliminates the need for the user to remember the item or having to search for it in the products page again. By placing the users’ favourite products in one place, it also allows them to compare different items in the wish list side by side. This report begins by detailing the front end user experience of the wish list feature, then delves deeper into the making of this MVF describing the code and database developed throughout this project.

Front end:

When first entering the wish list, if the user is not logged in with an account then they cannot use the wish list feature. This ensures that the wish list is specific to a particular user as different users will have different favourite items.

Upon login, users will see a change in messages to inform that they now have access to the wish list feature. They will also notice a new button appear saying Clear Wishlist which allows them to remove the items from the wish list should they wish to remove the items they add later on.

Looking at a product page on the OES site for a Samsung TV, users will be able to add this item to their wish list by selecting the button Add to Wishlist. This Add to Wishlist button appears on every individual product page only if the user is logged in, like the Clear Wishlist button in the Wish List page.

Users will be notified if they have successfully added an item to the wish list.

Users will immediately see the items they have added under the Wish List page.

This is what a populated wish list looks like. Compared to our mockups of the OES site, we have stayed true to the grid layout of the items in the wish list. The only thing that was altered was that we had changed it to 5 columns instead of 3 as there were too many blank spaces.
